# 바하마 뉴프로비던스 디지털 트윈 구축 결과 보고서

바하마 나소(Nassau) 인근 해안을 배경으로 한 물리 시뮬레이션 통합 **기후변화 디지털 트윈 PoC** 개발을 완료했습니다.

## 주요 구현 사항 및 아키텍처

> [!TIP]
> **데이터베이스 최적화 (Base Data Integration)**
> 지오메트리 관점에서 발생할 수 있는 붕뜸 현상을 해결하기 위해 요청하신 대로 `Cesium World Terrain`과 해저 공간을 묘사하는 `Cesium World Bathymetry(CWB)`를 적용하였고, 이에 완벽히 호환되는 **`Cesium OSM Buildings for CWB`** 플러그인을 불러옵니다.
> 배경 영상은 기본값으로 Google Map Satellite를 사용하며, UI에서 Google Photorealistic 3D, Bing Maps Aerial 등으로 자유롭게 스위칭이 가능합니다.

### 1. 프리미엄 UI 및 웹 클라이언트 구축
- **최신 스택**: Vanilla JS와 Vite를 활용하여 빌드 및 핫리로드 속성을 최적화했습니다.
- **Glassmorphism 디자인**: 사용자의 기대를 충족시키기 위해, 화면을 가리지 않으면서도 미려함을 제공하는 반투명 유리 질감 패널을 좌측 상단에 배치했습니다. (부드러운 전환 효과, 모던 폰트 적용)

### 2. 시나리오 A: 맹그로브 홍수 저감 효과 (파이썬 시뮬레이션 기반)
> [!NOTE]
> 브라우저상에서의 유체 동역학 연산 부하를 막기 위해, 백엔드 차원에서 사전에 **천수 방정식(Cellular Automata) 기반 파이썬 시뮬레이터**를 구동하여 시계열 침수 데이터를 도출했습니다.

- **파이썬 스크립트 (`simulation.py`)**: 나소 지역 DEM과 대응하는 200x100 해안 격자를 생성하고 높이(Z값)와 마찰 개수(Flow coefficient)를 할당했습니다. 맹그로브 유무에 따라 해안에 도달한 폭풍 서지가 이동하는 속도를 다르게 연산하여 각각 25프레임의 이미지 마스크를 산출 및 저장했습니다. 
- **Cesium 연동**: `Cesium.ImageMaterialProperty(CallbackProperty)`를 통해 사용자가 UI에서 'Play Simulation'을 누르면 시계열 이미지가 자연스럽게 오버레이됩니다. 토글 버튼을 켜면(맹그로브 적용 시) 해일이 내륙으로 침범하지 못하고 해안가에서 마찰로 인해 에너지를 잃는 모습을 직관적으로 볼 수 있습니다.

### 3. 시나리오 B: 해수면 상승 시뮬레이션
- 연도별 기후변화 최악의 시나리오를 가정하기 위해, 사용자가 슬라이더(0m ~ 5m)를 조작할 수 있도록 했습니다.
- 현재 위치 기준 0m(MSL)에서 시작해 Z(고도)를 지정한 높이만큼 상승시키는 반투명 워터 레이어로 시각화됩니다. 

## 실행 방법 및 테스트
1. 프로젝트 루트(`bahamas-digital-twin`)에서 `npm install` 및 `npm run dev`를 실행합니다.
2. 로컬호스트 주소를 접속하여 좌측 프리미엄 컨트롤 패널을 통해 두 가지 시나리오를 시연해볼 수 있습니다. 

> [!IMPORTANT]
> 1. 시뮬레이션 데이터인 `sim_no_mangrove` 멏 `sim_with_mangrove`의 폴더 내 데이터 수집에 필요한 `uv` 및 파이썬 시뮬레이터 연산 절차가 마무리되었습니다.
> 2. 실제 Cesium Ion Token이 필요한 유료 애셋(Google 3D 등) 환경 배포 시, `main.js` 파일에 토큰 값을 추가로 부여하면 제약 없이 렌더링이 이루어집니다.
